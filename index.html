Up until yeasterday, there were only Java and Jruby drivers for ADS.
Now we have one for Ruby too.

Motivation

why we decided to do it?
first of all we need to explain - when you say Ruby, it actually refers to multiple implementations
one reference, Matz Ruby Interpreter or MRI, and many others with Different feature sets
at the moment, we are dependent on the jruby implementation, which is based on top of JVM
since it is the only implementation with bindings for an ADS client
we don't really like this a lot, because we know that other interpreters have their own strengths
as is apparent from the graph
and we also feel that development without JVM would be much more convenient

How we did it

how we did it
There is already a java server in place written on top of Apache Thrift
all we had to do was write a client on top of the Thrift protocol

What can it do

So to the point - what can the driver actually do?
  • execute queries on the database (SELECT..)
  • execute statements on the database (UPDATE, INSERT, CREATE TABLE..)
  • fetch results with metadata (column names, types..)

How it helped us and everyone

Now the most important part - how this project helped us and everyone else
  • no need to use JVM in local development
  • faster response loop
  • endless performance optimalization possibilities
  • more platform possibilities
no need to use JVM locally - or actually anywhere else much faster response loop endless performance optimalization possibilities, as now can use every ruby interpreter and pick the right one for the job and finally the posibility to run on even more platforms
